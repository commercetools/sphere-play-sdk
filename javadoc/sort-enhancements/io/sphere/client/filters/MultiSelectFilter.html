Interface MultiSelectFilter<T>

  • Type Parameters:
    T - Type of individual values that can be selected by the user (e.g. String, Double, BigDecimal, Range<Double> etc.).
    All Superinterfaces:

    All Known Implementing Classes:
    Filters.NumberAttribute.Selectable.Ranges, Filters.NumberAttribute.Selectable.Values, Filters.Price.Selectable.Ranges, Filters.Price.Selectable.Values, Filters.StringAttribute.Selectable.Values, MultiSelectFilterBase

    public interface MultiSelectFilter<T>
extends Filter
    Filter 'component' that supports user selection from multiple predefined values and keeping the state of selected values in application's URL query string. See FilterParser for reconstructing state of multiple filters from application's URL. Use {#getUrlParams} to construct URLs for selecting individual filter values. You can also use helper methods {#getSelectLink}, {#getUnselectLink} and {#isSelected} that make use of {#getUrlParams}.

    Note: This class is quite similar to Facet. The difference is that filters simply filter results while facets do filtering and also result count aggregation. See also: UserInputFilter
